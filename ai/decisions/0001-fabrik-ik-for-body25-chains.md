# ADR 0001: IK для цепей BODY_25 через FABRIK

## Статус

Принято

## Контекст

В режиме IK нужно перемещать конечности (кисти, стопы) по цепочке суставов с сохранением длин костей. Классические Jacobian/CCD возможны, но для коротких цепочек (2 сегмента у руки/ноги) FABRIK даёт предсказуемое поведение и устойчивость. OpenPose экспортирует только координаты точек — внутренняя модель остаётся векторной в `three`.

## Решение

- Использовать **FABRIK** (`src/lib/solvers/FABRIKSolver.ts`) для четырёх IK-цепей, объявленных в `IKChains.ts` (руки и ноги до запястий/лодыжек).
- Конечные эффекторы: `IK_END_EFFECTORS`; сустав вне цепи в IK-режиме падает обратно на FK (`PoseService`).
- Сходимость и длины костей покрыты unit-тестами солвера.

## Последствия

- Смена алгоритма IK потребует нового ADR и регрессионных тестов в `fabrik.test.ts`.
- Ограничения углов (joint limits) пока не встроены в FABRIK — планируется отдельно (`refactoring-plan-design-doll-controls`, фаза A2).
