# ADR 0002: Проекция экспорта PNG/JSON и согласование с viewport

## Статус

Принято

## Контекст

Экспорт позы в PNG и OpenPose JSON должен визуально соответствовать тому, что пользователь видит в канвасе. Искажение NDC при несовпадении соотношения сторон камеры и целевого изображения даёт «сломанный» скелет. Обрезка по рамке экспорта должна совпадать с интерактивной рамкой в UI.

## Решение

- **Проекция 3D→2D**: `ExportService.projectTo2D` через `Vector3.project(camera)` и перевод NDC в пиксели с размерами, согласованными с aspect камеры (как WebGL viewport).
- **Линии вне кадра**: отрезки обрезаются по прямоугольнику кадра алгоритмом **Liang–Barsky** (`clipLineToRect` в `src/lib/utils/geometry.ts`).
- **Crop-экспорт**: `downloadPNGWithCrop` рендерит во внутреннем разрешении с тем же aspect, что viewport, затем вырезает область рамки (`PLAN` / `ExportFrame`).

## Последствия

- Любое изменение размеров канваса или логики рамки экспорта должно проверяться на согласованность с `ExportService` и при необходимости обновлять этот ADR.
- Юнит-тесты покрывают часть логики JSON без DOM; полная визуальная проверка — вручную.
