# R3F smoke/manual чек-листы PoseFlow

Этот документ покрывает то, что unit-тесты не видят: React Three Fiber сцену, pointer events, OrbitControls, гизмо, визуальную синхронизацию и экспорт из текущего viewport.

Используй его после изменений в:

- `poseflow/src/components/Canvas3D.tsx`
- `poseflow/src/components/controllers/`
- `poseflow/src/components/skeleton/`
- `poseflow/src/hooks/*Drag*.ts`
- `poseflow/src/services/RigService.ts`, если меняется пользовательское управление позой
- `poseflow/src/services/ExportService.ts`, если меняется проекция, crop или PNG export

Перед ручной проверкой обязательно должен пройти `npm run verify` из `poseflow/`, если задача не помечена как docs-only.

---

## 1. Подготовка

Запуск:

```bash
cd poseflow
npm run dev
```

Открыть: `http://127.0.0.1:5173`

Перед началом:

- [ ] В браузере нет красного runtime error overlay.
- [ ] В консоли нет повторяющихся ошибок React/R3F/Three.js.
- [ ] Скелет BODY_25 виден в viewport.
- [ ] Сетка/оси отображаются согласно настройкам.
- [ ] OrbitControls работают: middle drag pan, right drag rotate, wheel zoom.

## 2. Быстрый smoke-чеклист

Выполняется после каждой UI/R3F фичи. Цель — за 5-7 минут поймать поломки сцены.

### Viewport

- [ ] Сцена не пустая: видны кости и суставы.
- [ ] Камера не улетает после первого render.
- [ ] Поворот/zoom/pan камеры работают без зависания.
- [ ] Клик в пустое место снимает выделение.

### Selection

- [ ] Клик по MID_HIP/бедру выделяет pelvis.
- [ ] Клик по NECK/плечу выделяет spine.
- [ ] Клик по NOSE выделяет neck.
- [ ] Клик по глазу/уху выделяет head.
- [ ] Клик по локтю/запястью выделяет arm_r или arm_l.
- [ ] Выделенный элемент подсвечивается, остальные суставы остаются читаемыми.

### Controllers

- [ ] У pelvis видны 3 стрелки и 3 кольца.
- [ ] У spine видны кольца изгиба/скручивания.
- [ ] У neck видны кольца изгиба/скручивания.
- [ ] У head видны кольца yaw/pitch/roll.
- [ ] У arm видна сфера на запястье и дуга локтя.
- [ ] Drag любого активного контроллера меняет позу без runtime errors.
- [ ] Во время drag OrbitControls не перехватывают движение камеры.

### History

- [ ] После одного drag `Ctrl+Z` возвращает позу назад.
- [ ] `Ctrl+Y` или `Ctrl+Shift+Z` возвращает изменение.
- [ ] Один drag считается одним undo-шагом, а не десятками микрошагов.

### Export frame

- [ ] Кнопка Export Frame показывает рамку.
- [ ] Рамку можно двигать и resize-ить за ручки.
- [ ] Внутри рамки редактируется рамка, снаружи сохраняется возможность позировать.
- [ ] Export не падает, если камера уже зарегистрирована.

## 3. Полный R3F regression-чеклист

Выполняется перед закрытием новой R3F/controller фичи или перед релизной сборкой.

### Камера и viewport

- [ ] Все 9 видов в CameraControls переключают камеру.
- [ ] Reset возвращает камеру в ожидаемый вид.
- [ ] После переключения вида drag-контроллеры всё ещё работают.
- [ ] Нет визуального наложения overlay/кнопок на критичные элементы viewport.
- [ ] Размер текста в overlay не ломает layout.

### Pelvis

- [ ] X/Y/Z стрелки двигают таз по ожидаемым мировым осям.
- [ ] X/Y/Z кольца вращают позу вокруг ожидаемых осей.
- [ ] Undo/redo после translate работает.
- [ ] Undo/redo после rotate работает.
- [ ] После mirror pelvis-контроллер остаётся доступным.

### Spine

- [ ] Twist ограничен визуально разумным диапазоном, без переворота тела.
- [ ] Bend forward/back меняет дугу позвоночника.
- [ ] Lateral bend меняет дугу позвоночника.
- [ ] Виртуальные сегменты spine отрисованы непрерывной дугой.
- [ ] Undo/redo сохраняет virtual chain состояние.

### Neck

- [ ] Twist шеи работает отдельно от головы.
- [ ] Forward/back bend меняет дугу шеи.
- [ ] Lateral bend меняет дугу шеи.
- [ ] NECK→NOSE отрисован сегментами без разрыва.
- [ ] Undo/redo возвращает состояние шеи.

### Head

- [ ] Yaw вращает жёсткий блок головы вокруг NECK.
- [ ] Pitch имеет асимметричный лимит: назад меньше, вперёд больше.
- [ ] Roll наклоняет глаза/уши/нос как единый блок.
- [ ] Глаза и уши не расходятся относительно носа.
- [ ] Undo/redo возвращает голову.

### Arms

- [ ] Wrist sphere двигает запястье camera-plane drag.
- [ ] Shoulder остаётся фиксированным при wrist IK.
- [ ] Elbow bend решается без растяжения плечо→локоть и локоть→запястье.
- [ ] Elbow twist вращает локоть вокруг оси shoulder→wrist.
- [ ] Левая и правая руки работают симметрично.
- [ ] После mirror обе руки остаются управляемыми.

### Settings

- [ ] Открытие/закрытие SettingsModal не ломает управление камерой.
- [ ] Размер контроллеров меняется и не ломает hit areas.
- [ ] Изменение скорости OrbitControls применяется без reload.
- [ ] Тема/цветовая схема canvas не скрывает кости/суставы.

### Mini-view

Если включён `USE_MINI_VIEW`:

- [ ] Mini-view отображается в правом нижнем углу.
- [ ] Mini-view не перекрывает основные контроллеры.
- [ ] При вращении основной камеры mini-view остаётся синхронизированным.
- [ ] Mini-view не перехватывает pointer events основного viewport.

### Export

- [ ] PNG export даёт чёрный фон и цветные OpenPose кости/суставы.
- [ ] Export frame crop соответствует видимой рамке.
- [ ] JSON export содержит 25 BODY_25 точек по 3 значения.
- [ ] После поворота камеры PNG соответствует текущему viewport aspect.

## 4. Признаки R3F-регрессии

Если обнаружено что-то из списка, задачу нельзя считать завершённой без явной пометки риска:

- Canvas стал пустым или чёрным без скелета.
- OrbitControls продолжают двигать камеру во время drag гизмо.
- Drag создаёт десятки undo-шагов за одно действие.
- Выделение элемента не соответствует клику по суставу.
- Гизмо видно, но его hit area не реагирует.
- После `Ctrl+Z` визуальная поза и данные экспорта расходятся.
- Export PNG не совпадает с видимой рамкой/камерой.
- В консоли появляются повторяющиеся R3F warnings или runtime errors.

## 5. Формат отчёта

В финальном ответе по R3F/UI задаче указывать:

```text
Manual R3F:
- Smoke: passed / failed / not run
- Regression: passed / partial / not run
- Checked scenarios: ...
- Known issues: ...
```

Если manual check не запускался, указать причину: нет dev server, задача docs-only, нет доступа к browser, или пользователь не просил визуальную проверку.
