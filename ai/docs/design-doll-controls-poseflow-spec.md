# Управление по образцу DesignDoll для PoseFlow Editor

> [REFERENCE] UX/domain reference. Перед использованием сверить с текущей
> rotation-tree архитектурой в коде, `STATUS.md` и `PLAN.md`.

Документ основан на официальных туториалах Terawell DesignDoll ([страница со списком](https://terawell.net/en/page-tutorials.php)) и дополнен требованиями к **PoseFlow**: скелет OpenPose BODY_25, фиксированные длины костей (без «растяжения»), отдельная гибкость **позвоночника и шеи**, жёсткий **череп**, биомеханические **ограничения** на вращения и отсутствие вращения у части узлов, **семь главных контроллеров** (аналог ключевых точек управления DesignDoll).

Ниже — **10 разделов**, по одному на каждую ссылку с той страницы: сначала суть из DesignDoll, затем **что переносим в PoseFlow** и **что меняем** с учётом специфики.

## Общие принципы PoseFlow (целевая модель)

- **Формат**: 25 ключевых точек BODY_25; кости — отрезки фиксированной длины между родителем и потомком в графе (как сейчас в `SkeletonGraph`), без изменения длины при обычном позинге.
- **Исключение по «гибкости»**: вводятся **дополнительные степени свободы** только для **цепочки шеи и позвоночника** (см. виртуальные сегменты между существующими точками или расширение графа), чтобы имитировать наклоны/скручивание торса; остальные сегменты — жёсткие шарниры с ограничениями.
- **Голова / череп**: точки лица и головы (NOSE, глаза, уши и т.д.) образуют **жёсткую группу** — движутся как единый блок относительно шеи; отдельного «гнуть череп» нет.
- **Семь контроллеров** (главные манипуляторы, по аналогии с «бедром / кистями / стопами» и ключевыми точками торса в DesignDoll):
  1. **Основание черепа** (соединение с шеей; опорная точка для жёсткой головы).
  2. **Верхняя треть позвоночника** (верхний торс; условная точка на цепи позвоночника).
  3. **Крестец** (низ поясничного отдела / тазовый пояс; у BODY_25 ближе всего к **MID_HIP** и области таза).
  4–5. **Основания кистей** — **LEFT_WRIST**, **RIGHT_WRIST**.
  6–7. **Лодыжки** — **LEFT_ANKLE**, **RIGHT_ANKLE**.

Остальные суставы задаются **производно** (IK/FK, ограничения), а не отдельными «главными» гизмо, как в полном DesignDoll.

- **Шарниры без вращения / с лимитами**: таблица допустимых DOF задаётся по анатомии (например, колено — в основном сгибание в одной плоскости; тазобедренный — сгибание/разведение с клиньями). Узлы вроде **MID_HIP** как чисто «корневой» якорь могут не иметь пользовательского вращения — только перенос через контроллер крестца/таза.

- **Мини-вид**: в DesignDoll отдельное окно даёт **глубину** для рук/ног; в PoseFlow это должно соответствовать **mini-view на 90°** (как в плане Step 8).

## Шаг 1 — [PoseTag](https://terawell.net/en/page-tutorials-PoseTag.php) (создание позы, основной режим)

**Что описывает DesignDoll**

- Порядок работы: **1) торс → 2) плечи → 3) кисти и стопы → 4) мелкая настройка**.
- **Торс**:
  - *Сгибание*: клик по **контроллеру бёдер (hip)** → перетаскивание бледных **квадрата или треугольника** вверх/вниз.
  - *Скручивание*: перетаскивание **красных/синих** зон или **самой модели груди** (не контроллера) — важно: в оригинале явно разделено «контроллер» vs «поверхность модели».
  - *Наклон всего тела*: **[Tab]** для режима стрелок → перетаскивание **стрелочных контроллеров**.
- **Плечи**: позиция меняется **прямым перетаскиванием частей изображения** (меша).
- **Кисти и стопы**: логика **«точка A + направление B»** (DesignDoll) в отличие от классической «ось A, ось B на 360°» в других 3D-пакетах. Сначала ставится контактная точка **A**, затем направление **B**; глубина в 2D основного вида компенсируется **мини-видом** справа снизу (вид «с земли» на 90° вправо от основного). **Бледное кольцо** вокруг контроллера кисти/стопы задаёт направление **B**. Варианты до/после 5.3 и «Simple Controller» — кольцевой выбор угла.
- **Микро-подстройка**: прямое перетаскивание головы, шеи, стоп, кистей; у частей появляются **контроллеры-кольца** для вращения по осям после выбора.

**Дополнительно на странице**: снятие **shoulder link / wrist link** (аналог наших unlink) для независимого движения плеч и запястий.

**Перенос в PoseFlow**

- Режим «Pose» ≈ единственный режим редактирования скелета (без тегов как в DD).
- **Торс**: вместо отдельного «hip gizmo» квадрат/треугольник — связка **контроллер крестца/таза (MID_HIP + крестец)** + **верхняя треть позвоночника** + сгибание цепи **шея–позвоночник** с лимитами; скручивание торса — не деформация меша, а **сумма поворотов** разрешённых сегментов позвоночника (и при необходимости IK цепей ног от таза).
- **Плечи**: прямой drag суставов плеча/локтя — у нас **camera-plane drag** + FK; «перетаскивание меша» заменяется **перетаскиванием сфер суставов**.
- **Кисти/стопы**: модель **A + B** переносится как: сначала позиция **контроллера** (запястье/лодыжка), **глубина** — мини-вид; направление **B** — **кольцевой гизмо** (как в плане Step 11) с учётом лимитов сустава.
- **Микро-подстройка**: мелкие суставы и **голова**: голова — только как часть **жёсткого черепа** от основания черепа; шея — в пределах flex-цепи.

## Шаг 2 — [MorphingTag](https://terawell.net/en/page-tutorials-MorphingTag.php) (морфинг тела и лица)

**Что описывает DesignDoll**

- Тег **Morphing**: выбор частей тела, смешивание пресетов форм лица/мышц, ползунки с процентами бленда; настройка чувствительности мыши в Preferences.

**Перенос в PoseFlow**

- **Не переносится** как редактирование меша: у нас **нет** морфинга геометрии тела.
- Для рефакторинга: раздел «морфинг» **исключить**; при необходимости в будущем — отдельный модуль масштаба/пресетов **только позы**, не формы тела.

## Шаг 3 — [ScaleTag](https://terawell.net/en/page-tutorials-ScaleTag.php) (масштаб частей тела)

**Что описывает DesignDoll**

- **Simple**: круги — длина, кольца — толщина.
- **Detail**: чёрные стрелки — длина; шар на кости — растяжение всей кости; кольца — толщина XY; шарики на кольцах — толщина по одной оси; стрелки XYZ — сдвиг начала/конца кости.

**Перенос в PoseFlow**

- Масштабирование **отдельных костей** в длину/толщину **не входит** в текущую цель (фиксированные длины для ControlNet-скелета).
- Имеет смысл оставить **глобальный scale всей фигуры** (как уже есть в `PoseService.scale`) как аналог «общего размера модели», но **не** покомпонентный ScaleTag.
- Рефакторинг: явно **отделить** «масштаб сцены/фигуры» от «редактирования пропорций DD».

## Шаг 4 — [HandTag](https://terawell.net/en/page-tutorials-HandTag.php) (руки: простой и детальный режим)

**Что описывает DesignDoll**

- Сначала **Simple** — грубая позиция кисти.
- Переключение в **Detail**: линии для углов пальцев, изгиб по «швам» суставов, стрелка на кончике пальца — расстояние между пальцами.
- Библиотека пресетов кистей; с версии 3.5 — тег Hand; **H** или двойной клик по руке в 3D.

**Перенос в PoseFlow**

- BODY_25 **не содержит** суставов пальцев — только запястье.
- **Переносится идея двух уровней**: **грубо** — позиция и ориентация через **контроллер запястья** (один из семи) + кольца направления; **детально** — либо **отложено**, либо в будущем отдельная схема (не BODY_25).
- Пресеты «формы руки» из библиотеки DD **не применимы** без расширения скелета.

## Шаг 5 — [ImportTag](https://terawell.net/en/page-tutorials-ImportTag.php) (импорт OBJ и привязка)

**Что описывает DesignDoll**

- Добавление тега Import, выбор mesh, размер, ось, привязка к части тела (по умолчанию правая кисть), позиция и угол в 3D.

**Перенос в PoseFlow**

- Импорт сторонних мешей в сцену **не является** частью поз-редактора для ControlNet.
- Если понадобится референс: отдельная задача (пропсы); к **управлению скелетом** не относится.

## Шаг 6 — [SketchTag](https://terawell.net/en/page-tutorials-SketchTag.php) (рисование на модели)

**Что описывает DesignDoll**

- Рисование вспомогательных линий на модели, толщина/цвет/прозрачность, режим «сквозь» модель; несколько слоёв Sketch; импорт 1024×1024 как UV.

**Перенос в PoseFlow**

- Не часть целевого управления скелетом; при необходимости — **отдельный слой** аннотаций в 2D/оверлей без связи с DD PoseTag.

## Шаг 7 — [ClayTag](https://terawell.net/en/page-tutorials-ClayTag.php) (деформация формы как «глина»)

**Что описывает DesignDoll**

- Кисть, деформация меша перетаскиванием; сила (Strength) и накопление тегов влияют на итог.

**Перенос в PoseFlow**

- Не переносится: у нас **скелет**, не скульптура меша.

**Что описывает DesignDoll**

## Шаг 9 — [Pose Creation (Practice)](https://terawell.net/en/page-tutorials-Practice.php) (практический сценарий позы)

**Что описывает DesignDoll (ключевые приёмы)**

- Выбор **Pose tag**, вращение камеры ПКМ, **перетаскивание груди** для скручивания верха тела.
- Наклон спины: **hip controller** + бледный **box** вниз (простой контроллер); в новых версиях — иной UI.
- Плечи: поднять/опустить; вес на плече — с **бокового** вида.
- Голова: «шина» вокруг головы — **светлое горизонтальное кольцо** поворот, **среднее вертикальное** — наклон к плечу.
- Руки: снять **shoulder link**, движение кисти к бедру; **мини-вид** для глубины; **бледное кольцо** для угла предплечья.
- Ноги: рамка выделения ЛКМ по верху тела и бёдрам; **красные контроллеры на мини-виде** — сдвиг по глубине для сидячей позы; прямое перетаскивание **колена** «как фигурку»; центр тяжести: квадратный контроллер на груди; **Ctrl+клик** по контроллеру бедра — исключить из выделения; совместный сдвиг верхней части для баланса.
- Финал: подправить лицо, запястья, лодыжки; **Hand tag** для кисти.

**Перенос в PoseFlow**

- Сценарий **тот же по логике**: торс (крестец + верхняя треть позвоночника + flex шея/спина) → плечи → **4 конечности через WRIST/ANKLE + кольца B** → баланс (center of gravity — см. `PLAN.md`, Step 10) → мелочи.
- Прямое «таскание колена» заменяется **IK/FK колена** с фиксированной длиной бедра/голени и лимитами.
- Выделение нескольких контроллеров + Ctrl — аналог **группового переноса** верхнего блока (агрегация контроллеров позвоночника/ребра в одном drag).

## Шаг 10 — [Introduction to Using Cameras](https://terawell.net/en/page-tutorials-UseCamera.php) (камера и перспектива)

**Что описывает DesignDoll**

- Окно **Camera view** (жёлтая стрелка / Alt+C / меню Camera), изменение размера окна.
- Точка зрения настраивается **как в 3D-виду**.
- Режимы **Pers**: Real Perspective; параметры **FOV**, **Distortion**, **Distance**; клик-перетаскивание по числовым полям.
- Рекомендации по FOV (игры 45–60°, иллюстрации и т.д.).
- **Fake Pers**: выбор центра перспективы на модели, зоны фокуса A/B, акцент C/D.

**Перенос в PoseFlow**

- Уже близко: **OrbitControls**, пресеты камеры (`cameraService`), экспорт с учётом FOV/вида.
- Fake Pers **не обязателен** для первой версии рефакторинга управления скелетом; приоритет — согласованность **viewport ↔ экспорт** (уже в `ExportService`).
- Для соответствия DD: явная панель **FOV / «расстояние»** к камере в UI (опционально позже).

## Примечание для плана рефакторинга

1. Выделить **слой «режим Pose»** (аналог Pose tag): ввод только скелета, без «тегов» DD.
2. Реализовать **семь контроллеров** как первичные манипуляторы; остальные точки BODY_25 — вторичные или IK-цепочки.
3. Ввести **модель сегментов позвоночника/шеи** (гибкость) поверх фиксированных длин «конечностей».
4. Зафиксировать **череп** как жёсткую систему от **основания черепа**.
5. Добавить **таблицу ограничений** суставов (запрет/клинья углов).
6. Повторить **логику A + B + мини-вид** для кистей и стоп.
7. Камера по мотивам шага 10 — постепенно, без блокировки скелета.
