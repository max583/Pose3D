# Анализ текущего поведения drag системы и план перехода к DesignDoll

## Текущее состояние drag системы

### 1. Архитектура
- **Хук**: `useTransformDrag` в `src/hooks/useTransformDrag.ts`
- **Принцип работы**: Drag в плоскости, перпендикулярной камере
- **Использование**: Компонент `Joint` использует хук для перетаскивания суставов

### 2. Ключевые характеристики текущей системы

#### Положительные аспекты:
- **Работа с любой камеры**: Плоскость перпендикулярна направлению камеры
- **Отсутствие "прыжков"**: Offset компенсирует разницу между курсором и суставом
- **Плавность**: Непрерывное обновление позиции во время drag
- **Изоляция**: Каждый сустав имеет независимый drag state

#### Ограничения (по сравнению с DesignDoll):
1. **Нет фиксированных длин костей**: Кости растягиваются/сжимаются при drag
2. **Нет жесткого черепа**: Голова деформируется как отдельные суставы
3. **Нет контроллеров**: Drag напрямую на суставы, а не на контроллеры
4. **Нет цепей позвоночника**: Позвоночник как отдельные суставы без группировки
5. **Нет виртуальных сегментов**: Только реальные BODY_25 суставы

### 3. Текущий flow drag операции
```
1. Пользователь кликает на сустав (Joint компонент)
2. Создается плоскость перпендикулярно камере через сустав
3. Вычисляется offset между курсором и суставом
4. При движении мыши:
   - Луч пересекает плоскость
   - Новая позиция = точка пересечения - offset
   - Вызывается onPositionChange с новой позицией
5. PoseService обновляет позу с учетом FK/IK
```

## Требования DesignDoll системы

### 1. Ключевые отличия DesignDoll
1. **Фиксированные длины костей**: Кости не растягиваются
2. **Жесткий череп**: Голова как недеформируемая группа
3. **7 контроллеров**: Основные точки управления вместо отдельных суставов
4. **Цепь позвоночника**: Виртуальные сегменты для плавного изгиба
5. **Двунаправленный маппинг**: Между виртуальным состоянием и BODY_25

### 2. Контроллеры DesignDoll (7 основных)
1. **Голова** (Head) - управляет всей головой
2. **Грудь** (Chest) - верхняя часть туловища
3. **Таз** (Pelvis) - нижняя часть туловища
4. **Правая рука** (Right Arm) - вся рука
5. **Левая рука** (Left Arm) - вся рука
6. **Правая нога** (Right Leg) - вся нога
7. **Левая нога** (Left Leg) - вся нога

### 3. Виртуальные сегменты
- **Позвоночник**: 5-7 виртуальных сегментов между тазом и головой
- **Шея**: 2-3 виртуальных сегмента между грудью и головой
- **Конечности**: Сохраняют реальные суставы BODY_25

interface VirtualPoseState {
  // Виртуальное состояние
  spine: SpineState;
  skull: SkullState;
  controllers: Map<ControllerId, ControllerState>;
  
  // Реальное состояние (для обратной совместимости)

  // Частичное обновление

}
